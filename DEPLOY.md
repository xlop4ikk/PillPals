# 🚀 Деплой «Пилюлькин День» — фоновые push-уведомления

Это руководство настраивает **бесплатные фоновые уведомления** (даже когда приложение закрыто) на iOS 16.4+ и Android. Используется **Cloudflare Workers + KV** — бесплатно и работает в РФ без VPN.

---

## Что нужно

1. **GitHub Pages** — для самого приложения (бесплатно, HTTPS).
2. **Cloudflare аккаунт** — для push-сервера (бесплатный тариф, регистрация без оплаты).

---

## Шаг 1. Залей приложение на GitHub Pages

1. Создай репозиторий (например `pillpals`).
2. Залей все файлы из корня проекта: `index.html`, `style.css`, `app.js`, `sw.js`, `manifest.json`, `icons/`, `.nojekyll`.
3. Settings → Pages → Source: ветка `main`, папка `/root`.
4. Дождись URL: `https://ТВОЙ-ЛОГИН.github.io/pillpals/`.

> Файл `.nojekyll` (пустой) нужен, чтобы GitHub Pages не ломал иконки через Jekyll.

---

## Шаг 2. Зарегистрируйся на Cloudflare

1. Зайди на https://dash.cloudflare.com/sign-up
2. Регистрация бесплатна, российская карта **не нужна** (бесплатный тариф не требует оплаты).
3. Подтверди email.

---

## Шаг 3. Сгенерируй VAPID-ключи

Ключи уже сгенерированы в `tools/gen_vapid.js`, но **для продакшена создай свои**:

```bash
node tools/gen_vapid.js
```

Вывод будет вида:
```
VAPID_PUBLIC_KEY (для клиента):      BD99Ubpc...
VAPID_PRIVATE_JWK (для Worker):      {"kty":"EC",...}
```

**Сохрани оба значения** — они понадобятся ниже.

> ⚠️ Приватный ключ — секрет! Не коммить его в публичный репозиторий.

---

## Шаг 4. Создай KV-хранилище

1. В дашборде Cloudflare: **Workers & Pages → KV → Create a namespace**.
2. Назови `PILLS`.
3. Скопируй **Namespace ID** (длинная строка).

---

## Шаг 5. Настрой и задеплой Worker

1. Установи Wrangler (CLI Cloudflare):
   ```bash
   npm install -g wrangler
   ```
2. Авторизуйся:
   ```bash
   wrangler login
   ```
   (Откроется браузер — войди в аккаунт Cloudflare.)

3. Открой `worker/wrangler.toml` и заполни:
   - `VAPID_PUBLIC_KEY` — твой публичный ключ
   - `VAPID_PRIVATE_JWK` — твой приватный ключ (одной строкой в кавычках)
   - `VAPID_SUBJECT` — твой email (`mailto:ты@example.com`)
   - В блоке `[[kv_namespaces]]` замени `id` на **Namespace ID** из шага 4.

4. Задеплой:
   ```bash
   cd worker
   wrangler deploy
   ```
   Wrangler выдаст URL вида: `https://pillpals-push.ТВОЙ-СУБДОМЕН.workers.dev`

---

## Шаг 6. Впиши URL Worker в приложение

Открой `app.js`, найди вверху:
```js
const PUSH_SERVER = "https://pillpals-push.YOUR-SUBDOMAIN.workers.dev";
const VAPID_PUBLIC_KEY = "BD99Ubpc...";
```
Замени на свой URL и свой публичный ключ. Залей обновление на GitHub Pages.

---

## Шаг 7. Проверь

1. Открой `https://ТВОЙ-ЛОГИН.github.io/pillpals/` на телефоне.
2. Разреши уведомления при первом тапе.
3. Добавь таблетку с временем через 2–3 минуты.
4. **Закрой приложение полностью** (смахни из списка).
5. Дождись времени — уведомление должно прийти из фона.

### Проверка здоровья сервера
Открой в браузере:
```
https://pillpals-push.ТВОЙ-СУБДОМЕН.workers.dev/api/health
```
Должно вернуть `{"ok":true,"time":"..."}`.

---

## Важно про iOS

Фоновый push на iPhone работает **только если**:
- iOS **16.4 или новее**.
- Приложение **установлено на главный экран** (Safari → Поделиться → «На экран Домой»).
- Уведомления **разрешены**.

Если открыть просто вкладкой в Safari — push не придёт из фона (ограничение Apple). Это нормально для всех PWA.

---

## Лимиты бесплатного тарифа Cloudflare

| Ресурс | Лимит | Достаточно для |
|--------|-------|----------------|
| Запросы Worker | 100 000/день | ~сотен пользователей |
| KV чтений | 100 000/день | cron + регистрация |
| KV записей | 1 000/день | обновления расписаний |
| KV хранилище | 1 ГБ | тысячи подписок |
| Cron triggers | бесплатно | каждые 2 минуты |

Для личного/семейного использования лимитов хватит с огромным запасом.

## Как это работает (схема)

```
 [Телефон]                         [Cloudflare Worker]           [Apple/Google]
    │                                      │                          │
    │── POST /api/register ───────────────▶│ сохраняет подписку       │
    │   (subscription + pills + tz)        │ + расписание в KV        │
    │                                      │                          │
    │                              cron каждые 2 мин                   │
    │                                      │── проверяет время ──────▶│
    │                                      │── шлёт push (VAPID+ECDH)▶│
    │◀──────── push уведомление ──────────────────────────────────────│
    │  (даже если приложение закрыто)       │                          │
```

Push-payload шифруется по RFC 8291 (aes128gcm) прямо в Worker через Web Crypto API — содержимое видит только устройство.
